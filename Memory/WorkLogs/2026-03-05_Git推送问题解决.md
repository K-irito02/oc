# Git 推送问题解决

## 问题描述
在尝试将代码提交到 GitHub 远程仓库时，遇到以下错误：
```
fatal: unable to access 'https://github.com/K-irito02/oc.git/': Recv failure: Connection was reset
fatal: unable to access 'https://github.com/K-irito02/qt-platform-app.git/': Failed to connect to 127.0.0.1 port 7890 after 2075 ms: Could not connect to server
```

## 问题分析
1. **代理配置问题**：Git 配置了本地代理（127.0.0.1:7890），但代理服务未运行或无法连接
2. **仓库 URL 错误**：平台代码仓库 URL 已变更为 `oc-platform-app`，但本地仍使用旧的 `qt-platform-app` URL

## 解决步骤

### 1. 移除 Git 代理配置
执行以下命令清除全局代理设置：
```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 2. 更新仓库 URL
对于平台代码仓库，更新远程仓库 URL：
```bash
git remote set-url origin https://github.com/K-irito02/oc-platform-app.git
```

### 3. 重新推送代码
```bash
# 推送环境配置仓库
cd e:\oc
git push origin main

# 推送平台代码仓库
cd e:\oc\oc-platform
git push origin main
```

## 结果
✅ 成功将代码推送到两个远程仓库：
- 环境配置仓库：`5e4095e..ea157b1`
- 平台代码仓库：`e1dab21..87541e8`

## 经验总结
1. 遇到 Git 连接问题时，优先检查代理配置
2. 定期验证远程仓库 URL 是否正确
3. 网络问题可能需要多次尝试或等待网络恢复
