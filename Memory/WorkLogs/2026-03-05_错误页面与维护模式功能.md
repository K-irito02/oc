# 错误页面与维护模式功能实现

> 日期: 2026-03-05

## 功能概述

实现了完整的错误页面系统和网站维护模式功能，包括：

### 1. 错误页面系统

#### 前端新增
- `src/pages/Error/index.tsx` - 通用错误页面组件
  - 404 页面未找到（蓝色主题，FileQuestion 图标）
  - 500 服务器错误（红色主题，ServerCrash 图标）
  - 403 权限错误（橙色主题，ShieldX 图标）
  - 网络错误（黄色主题，WifiOff 图标）

- 更新 `src/pages/NotFound/index.tsx` - 使用新的 Error 组件

#### 设计规范（基于 ui-ux-pro-max）
- 使用 Lucide React SVG 图标
- 支持深色/浅色主题
- 国际化支持（中英文）
- 响应式布局
- 脉冲动画效果
- WCAG AA 颜色对比度标准

### 2. 维护模式系统

#### 前端新增
- `src/pages/Maintenance/index.tsx` - 维护页面组件
  - 从后端获取维护配置
  - 显示维护标题、说明（中英文）
  - 预计恢复时间倒计时
  - 重新检查按钮

- `src/hooks/useMaintenance.ts` - 维护模式 Hook
  - 检查维护状态
  - 自动跳转到维护页面
  - 缓存维护状态

#### 后端新增
- `MaintenanceConfigDTO.java` - 维护配置 DTO
- `MaintenanceStatusDTO.java` - 维护状态 DTO
- `MaintenanceInterceptor.java` - 维护模式拦截器
  - 检查维护状态
  - 白名单路径（管理员可访问后台）
  - 返回 503 响应

#### API 接口
- `GET /api/v1/public/system/maintenance` - 获取维护状态（公开）
- `GET /api/v1/admin/system/maintenance` - 获取维护配置（管理员）
- `PUT /api/v1/admin/system/maintenance` - 更新维护配置（管理员）

#### 数据库配置
```sql
system.maintenance.enabled      -- 维护模式开关
system.maintenance.title        -- 维护标题（中文）
system.maintenance.title_en     -- 维护标题（英文）
system.maintenance.message      -- 维护说明（中文）
system.maintenance.message_en   -- 维护说明（英文）
system.maintenance.estimated_time -- 预计恢复时间
```

### 3. 管理后台

更新 `src/pages/Admin/System/index.tsx`：
- 维护模式开关（带警告提示）
- 维护标题输入（中英文）
- 维护说明文本域（中英文）
- 预计恢复时间选择器
- 配置保存后自动清除缓存

### 4. 路由配置

更新 `src/router/index.tsx`：
- `/maintenance` - 维护页面
- `/500` - 500 错误页面
- `/403` - 403 错误页面

### 5. HTTP 拦截器

更新 `src/utils/request.ts`：
- 403 响应 → 跳转 /403
- 500 响应 → 跳转 /500
- 503 响应 → 跳转 /maintenance

### 6. 国际化

更新 `src/locales/zh-CN.json` 和 `src/locales/en-US.json`：
- 错误页面翻译
- 维护页面翻译
- 管理后台维护配置翻译
