# 数据库系统配置修复

## 时间
2026-03-07

## 问题描述

### 问题现象
网站名称默认值不正确：
- **现状**：前端显示的网站名称为 `KiritoLab`
- **期望**：中文默认 `桐人创研`，英文默认 `KirLab`

### 问题原因
数据库中 `system_configs` 表是空的，没有系统配置数据。虽然前端默认配置正确，但后端 API 无法返回正确的网站名称。

## 解决方案

### 1. 检查数据库配置
```sql
SELECT config_key, config_value FROM system_configs WHERE config_key IN ('site.name', 'site.name_en');
-- 返回 0 rows
```

### 2. 重新插入系统配置数据
执行 SQL 插入系统配置：
```sql
INSERT INTO system_configs (config_key, config_value, description) VALUES
    ('site.name', '桐人创研', '站点中文名称'),
    ('site.name_en', 'KirLab', '站点英文名称'),
    -- ... 其他配置
ON CONFLICT (config_key) DO UPDATE SET config_value = EXCLUDED.config_value;
```

### 3. 验证配置
```sql
SELECT config_key, config_value FROM system_configs WHERE config_key IN ('site.name', 'site.name_en');
-- 返回:
--  config_key  | config_value 
-- -------------+--------------
--  site.name   | 桐人创研
--  site.name_en| KirLab
```

### 4. 重启后端服务
重启后端服务后，API 返回正确的配置：
```json
{
  "siteName": "桐人创研",
  "siteNameEn": "KirLab"
}
```

## 影响文件
- 数据库 `system_configs` 表

## 经验总结
1. 数据库初始化脚本 `init.sql` 中包含系统配置数据，但可能因为某些原因未正确执行
2. 需要确保数据库初始化时完整执行所有 INSERT 语句
3. 前端默认配置作为备用方案，但后端数据库配置是主要数据源

## 后续建议
1. 检查数据库初始化脚本执行日志
2. 考虑添加数据库配置检查脚本，在启动时验证必要配置是否存在
3. 文档化数据库初始化流程，确保新环境部署时配置完整
