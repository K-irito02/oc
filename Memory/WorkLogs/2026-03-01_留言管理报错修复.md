# 2026-03-01 留言管理报错修复

## 问题概述
- 管理后台留言管理页面无数据，接口返回 10000 未知错误。
- 前端构建时报错：Swiper 样式模块缺失、变量未使用、分页显示函数命名冲突等。

## 排查过程
1. 重启 Docker 依赖、后端与前端服务，确认基础服务正常。
2. 通过接口测试复现：登录获取 Token 后调用 `/api/v1/feedbacks/admin` 返回 10000。
3. 查看后端日志，确认是 `AccessDeniedException` 被全局异常处理器包装成未知错误。
4. 检查数据库角色：管理员账号实际角色为 `SUPER_ADMIN`。
5. 定位控制器权限注解：管理员留言接口仅允许 `ADMIN` 角色。
6. 检查前端构建错误：`showTotal` 参数与 `t` 翻译函数重名、Swiper CSS 模块缺声明、Logo 裁剪组件存在未使用变量。

## 根因
管理员账号是 `SUPER_ADMIN`，但留言管理接口只允许 `ADMIN`，导致方法级权限拒绝访问，最终被全局异常处理器返回未知错误。

## 修复方式
- 调整留言管理接口权限为 `ADMIN` 或 `SUPER_ADMIN`。
- 修复前端构建报错：
  - `showTotal` 回调参数改名，避免与翻译函数冲突。
  - 移除未使用的 `Space` 和 `fileInputRef`。
  - 在 `vite-env.d.ts` 中补充 Swiper CSS 模块声明。

## 验证结果
- `/api/v1/feedbacks/admin` 返回 `code=0` 且包含数据。
- `npm run build` 前端构建成功。
- `npm run lint` 仍存在历史遗留 lint 问题（与本次修复无直接关系）。
