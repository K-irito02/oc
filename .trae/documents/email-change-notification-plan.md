# 邮箱更改通知旧邮箱功能实施计划

## 概述

当前邮箱更改流程只发送验证码到新邮箱，缺少对旧邮箱的通知。根据用户需求，无论更改成功或失败，都需要发送通知邮件到旧邮箱，这是安全最佳实践。

## 当前流程分析

```
用户输入新邮箱 → 验证码发送到新邮箱 → 用户输入验证码 → 验证并更改邮箱
```

**问题**：旧邮箱没有任何通知，用户可能不知道邮箱被更改。

## 目标流程

```
用户输入新邮箱 → 验证码发送到新邮箱 → 用户输入验证码 → 验证并更改邮箱
                                                              ↓
                                              发送通知到旧邮箱（成功/失败）
```

## 实施方案

### 1. 新增邮件类型

在 `EmailVerification` 和相关服务中添加新的邮件类型：
- `EMAIL_CHANGE_SUCCESS` - 邮箱更改成功通知
- `EMAIL_CHANGE_FAILED` - 邮箱更改失败通知

### 2. 修改 AuthService.changeEmail 方法

在 `changeEmail` 方法中：
1. 获取用户的旧邮箱地址
2. 无论成功或失败，都发送通知邮件到旧邮箱
3. 通知邮件包含：
   - 操作时间
   - 新邮箱地址（部分隐藏）
   - 操作结果（成功/失败）
   - 失败原因（如果失败）

### 3. 新增邮件模板

在 `EmailTemplateService` 中添加：
- `generateEmailChangeNotificationEmail()` - 生成邮箱更改通知邮件

### 4. 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `AuthService.java` | 修改 `changeEmail` 方法，添加旧邮箱通知逻辑 |
| `EmailTemplateService.java` | 添加邮箱更改通知邮件模板生成方法 |
| `EmailService.java` | 添加发送邮箱更改通知的方法 |
| `EmailConfigService.java` | 确认配置完整性 |

### 5. 实施步骤

#### 步骤 1：修改 EmailTemplateService

添加新方法生成邮箱更改通知邮件：
- 支持成功和失败两种类型
- 包含操作时间、新邮箱（脱敏）、失败原因等信息
- 保持与现有邮件模板一致的设计风格

#### 步骤 2：修改 EmailService

添加发送邮箱更改通知的方法：
- `sendEmailChangeNotification(String oldEmail, String newEmail, boolean success, String reason)`

#### 步骤 3：修改 AuthService.changeEmail

重构 `changeEmail` 方法：
1. 在方法开始时保存旧邮箱地址
2. 使用 try-catch-finally 或其他方式确保无论成功失败都发送通知
3. 传递适当的失败原因

### 6. 邮件内容设计

#### 成功通知邮件

```
主题：【站点名称】邮箱更改成功通知

正文：
您的账户邮箱已成功更改。

原邮箱：user@old.com
新邮箱：u***r@new.com
更改时间：2026-03-08 12:34:56

如果这不是您本人的操作，请立即联系管理员。
```

#### 失败通知邮件

```
主题：【站点名称】邮箱更改尝试通知

正文：
有人尝试更改您的账户邮箱，但操作失败。

原邮箱：user@old.com
尝试更改的邮箱：u***r@new.com
尝试时间：2026-03-08 12:34:56
失败原因：验证码错误 / 邮箱已被占用 / 其他原因

如果这不是您本人的操作，请确保您的账户安全。
```

### 7. 安全考虑

1. **邮件地址脱敏**：在通知邮件中，新邮箱地址需要部分隐藏（如 `u***r@example.com`）
2. **异步发送**：通知邮件应该异步发送，不影响主流程响应时间
3. **频率限制**：考虑是否需要对通知邮件添加频率限制，防止滥用

### 8. 验证清单

- [ ] 邮箱更改成功时，旧邮箱收到成功通知
- [ ] 邮箱更改失败时（验证码错误），旧邮箱收到失败通知
- [ ] 邮箱更改失败时（邮箱已存在），旧邮箱收到失败通知
- [ ] 邮件内容包含正确的操作时间和脱敏的新邮箱
- [ ] 邮件模板支持中英文双语
- [ ] 异步发送不影响主流程性能
