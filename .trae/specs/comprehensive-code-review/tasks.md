# 安全修改原则

1. **不删除现有功能代码**：只修复问题，不删除功能实现
2. **保持 API 兼容性**：后端 API 接口签名和行为保持不变
3. **保持数据库兼容性**：不删除表、字段，不破坏现有数据
4. **保持前端兼容性**：组件接口和行为保持不变
5. **渐进式修复**：每次修改后验证功能正常再继续
